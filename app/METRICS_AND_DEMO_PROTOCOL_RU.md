# Методика замеров и протокол демо

## Метрики, которые фиксируем на защите

- **Сообщения:** время доставки (send->ACK), retry count, drop duplicates.
- **Сообщения:** время доставки (send->ACK), retry count, drop duplicates, `avg/p95 ACK RTT`, `loss%`.
- **Сеть:** количество узлов, количество рёбер, стабильность connected endpoints.
- **Файлы:** количество чанков, процент полученных чанков, checksum ok/fail.
- **Файлы:** количество чанков, процент полученных чанков, checksum ok/fail, сохранение частично полученных чанков после рестарта.
- **Интерактивность UI:** отсутствие freeze при отправке сообщений/файлов.

## Как измерять

1. Включить 2-3 устройства в mesh.
2. Отправить 20 текстовых сообщений с интервалом 300-500 мс.
3. Для каждого сообщения зафиксировать в логах время появления `send -> ...` и `ACK for ...`.
4. Посчитать p50/p95 по задержке подтверждения.
5. Запустить передачу файла и зафиксировать:
   - число чанков,
   - время до полной сборки,
   - результат checksum.
6. Для сценария потерь отключить ретранслятор посередине теста и показать:
   - рост retry,
   - восстановление после reconnection.

## Встроенные fault-инструменты (вкладка Dev)

- `Relay ON/OFF` — включение/отключение форвардинга мультихоп.
- `Loss 0/10/30%` — симуляция потерь inbound/outbound пакетов.
- `Reset metrics` — сброс счётчиков перед новым прогоном.
